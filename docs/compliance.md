# EU AI Act Compliance

Guide for checking your agents against EU AI Act requirements using `agentshift compliance`.

---

## Usage

```bash
agentshift compliance ./my-agent --from openclaw --framework eu-ai-act
```

---

## What it checks

AgentShift evaluates your agent against five articles of the EU AI Act:

| Article | Requirement | What AgentShift checks |
|---|---|---|
| **Art. 9** — Risk management | Risk mitigation measures in place | Guardrails list in skill definition |
| **Art. 10** — Data governance | Training/knowledge data is documented | Knowledge sources declared |
| **Art. 13** — Transparency | System is clearly described | Description field present and substantive |
| **Art. 14** — Human oversight | Humans can intervene | Human oversight mentioned in instructions |
| **Art. 52** — Disclosure | Users know they're talking to AI | "I am an AI" or equivalent disclosure in system prompt |

## How to pass each check

### Art. 9 — Risk management

Add a guardrails list to your skill definition:

```markdown
## Guardrails
- Never execute destructive commands without confirmation
- Refuse requests that violate company data policies
- Escalate to human when confidence is below threshold
```

### Art. 10 — Data governance

Declare knowledge sources:

```markdown
## Knowledge
- internal-docs/api-reference.md
- internal-docs/runbook.md
```

### Art. 13 — Transparency

Add a clear, substantive description to your agent:

```markdown
# my-agent

This agent monitors GitHub repositories for open pull requests and provides CI status summaries. It uses the `gh` CLI and has read-only access to repository metadata.
```

### Art. 14 — Human oversight

Mention human oversight in your instructions:

```markdown
## Human Oversight
All destructive actions (merge, close, delete) require explicit human approval before execution.
```

### Art. 52 — Disclosure

Add an AI disclosure to your system prompt:

```markdown
## Disclosure
I am an AI assistant. My responses are generated by a language model and should be reviewed by a human before acting on them.
```

## JSON output for CI integration

```bash
agentshift compliance ./my-agent --from openclaw --framework eu-ai-act --output-format json
```

Returns:

```json
{
  "agent": "my-agent",
  "framework": "eu-ai-act",
  "results": [
    { "article": "Art. 9", "status": "warn", "message": "No guardrails list found" },
    { "article": "Art. 10", "status": "pass", "message": "3 knowledge sources declared" },
    { "article": "Art. 13", "status": "pass", "message": "Description present (128 chars)" },
    { "article": "Art. 14", "status": "fail", "message": "No human oversight mention" },
    { "article": "Art. 52", "status": "fail", "message": "No AI disclosure in prompt" }
  ],
  "summary": { "pass": 2, "warn": 1, "fail": 2 }
}
```

Use in CI pipelines:

```yaml
- name: Compliance check
  run: |
    result=$(agentshift compliance ./my-agent --from openclaw --framework eu-ai-act --output-format json)
    fails=$(echo "$result" | jq '.summary.fail')
    if [ "$fails" -gt 0 ]; then
      echo "Compliance check failed with $fails violations"
      exit 1
    fi
```
