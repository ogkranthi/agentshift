# OpenClaw → Microsoft 365 Copilot

Full end-to-end guide for converting an OpenClaw skill to a Microsoft 365 Declarative Agent.

---

## Step 1: Convert

```bash
agentshift convert ~/.openclaw/skills/github --from openclaw --to m365 --output ./github-m365
```

Output:

```
github-m365/
├── declarative-agent.json    ← M365 Declarative Agent manifest
├── manifest.json             ← Teams App manifest (wraps the agent for sideloading)
└── README.md                 ← prerequisites + sideload instructions
```

---

## Step 2: Prerequisites

- Microsoft 365 license with Copilot enabled
- Teams Admin Center access (or Developer mode enabled in Teams)
- Microsoft 365 Agents Toolkit (VS Code extension) — optional but recommended
- App registration in Azure AD if the agent calls external APIs

---

## Step 3: Sideload

**Option A — Teams Admin Center:**

1. Zip the output folder: `cd github-m365 && zip -r ../github-m365.zip .`
2. Go to [Teams Admin Center](https://admin.teams.microsoft.com) → **Teams apps** → **Manage apps**
3. Click **Upload** and select `github-m365.zip`
4. Assign the app to users or groups

**Option B — M365 Agents Toolkit (VS Code):**

1. Open the output folder in VS Code
2. Install the [Microsoft 365 Agents Toolkit](https://marketplace.visualstudio.com/items?itemName=TeamsDevApp.ms-teams-vscode-extension) extension
3. Click **Provision** → **Deploy** in the Toolkit sidebar

---

## Step 4: Use

1. Open [Microsoft 365 Copilot](https://m365.cloud.microsoft/chat)
2. Click the agent picker (top-right) and select your agent
3. Start chatting — the agent's instructions are now active in your Copilot session

---

## What carries over from OpenClaw

| OpenClaw feature | M365 equivalent | Status |
|---|---|---|
| Skill instructions (body) | `declarative-agent.json` instructions field | ✅ Full fidelity |
| Conversation starters | `conversation_starters` in declarative agent | ✅ Mapped |
| Shell tools (gh, curl, etc.) | Dropped — M365 doesn't support shell execution | ❌ |
| MCP tools | API plugin (manual wiring required) | ⚠️ Manual implementation required |
| Knowledge files | SharePoint / OneDrive knowledge source (stub) | ⚠️ Manual setup required |
| Cron / scheduled triggers | Not supported | ❌ |
| Telegram / Slack delivery | Not supported | ❌ |

---

## Limitations

- **No shell tools** — M365 Declarative Agents run in the browser/Teams context; there is no shell execution environment
- **No cron / scheduled triggers** — declarative agents are invoked interactively only
- **API plugins require Azure AD app registration** — if the agent needs to call external APIs, you must register an app and configure OAuth
- **No Telegram/Slack delivery** — M365 agents respond within the M365 Copilot UI only

---

agentshift diff ~/.openclaw/skills/github --from openclaw --targets m365
