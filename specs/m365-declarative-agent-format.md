# Microsoft 365 Declarative Agent Format

## Overview

Microsoft 365 Declarative Agents are Copilot extensions defined via JSON manifests. They extend
Microsoft 365 Copilot with custom instructions, capabilities, and conversation starters —
without requiring custom code or plugins.

AgentShift emits two files for this platform:

| File | Purpose |
|------|---------|
| `declarative-agent.json` | Agent manifest with instructions and capabilities |
| `manifest.json` | Teams app manifest referencing the declarative agent |
| `README.md` | Deployment instructions |
| `instructions-full.txt` | (Optional) Full instructions when truncated to 8,000 chars |

---

## declarative-agent.json

### Schema

```
$schema: https://developer.microsoft.com/json-schemas/copilot/declarative-agent/v1.4/schema.json
version: v1.4
```

### Fields

| Field | Type | Limit | Source |
|-------|------|-------|--------|
| `name` | string | — | `ir.name` |
| `description` | string | 1,000 chars | `ir.description` |
| `instructions` | string | 8,000 chars | `ir.persona.system_prompt` |
| `capabilities` | array | — | Mapped from IR tools/knowledge |
| `conversation_starters` | array | max 6 items | Extracted from system_prompt |

### Capability Mapping

IR tools and knowledge are mapped to M365 capabilities as follows:

| IR Source | M365 Capability |
|-----------|----------------|
| `tools[kind=mcp, name=teams]` | `{"name": "TeamsMessages"}` |
| `tools[kind=mcp, name=email]` | `{"name": "Email"}` |
| `tools[kind=mcp, name=notion]` | `{"name": "GraphConnectors", "connections": []}` |
| `tools[kind=mcp, name=graph]` | `{"name": "GraphConnectors", "connections": []}` |
| `knowledge[kind=url]` | `{"name": "WebSearch", "sites": [{"url": path}]}` |
| `tools[kind=shell]` with curl/web | `{"name": "WebSearch"}` |
| `tools[kind=shell]` (other) | Dropped — TODO comment in manifest |

Shell tools without web access are **dropped** with a `# TODO` comment since M365 Declarative
Agents do not have shell execution capability.

### Conversation Starters

Up to 6 conversation starters are extracted from the system prompt by scanning for bullet
points (`-` or `*`) under sections titled "When to Use", "Examples", or "Try". Each starter is:

```json
{
  "title": "Short label (≤ 50 chars)",
  "text": "Full prompt text"
}
```

If no structured starters are found, the field is omitted.

### Instruction Truncation

If `ir.persona.system_prompt` exceeds 8,000 characters:

1. Truncate at the last sentence boundary before character 7,800
2. Append: `[AGENTSHIFT: truncated to 8,000 char M365 limit. Full text in instructions-full.txt]`
3. Write the full untruncated text to `instructions-full.txt`

## manifest.json

### Schema

```
$schema: https://developer.microsoft.com/json-schemas/teams/v1.17/MicrosoftTeams.schema.json
manifestVersion: 1.17
```

### Fields

| Field | Value |
|-------|-------|
| `id` | UUID v4 (generated per emission) |
| `name.short` | `ir.name` |
| `name.full` | `ir.description[:100]` |
| `description.short` | `ir.description[:80]` |
| `description.full` | `ir.description[:4000]` |
| `icons.color` | `"color.png"` (placeholder) |
| `icons.outline` | `"outline.png"` (placeholder) |
| `accentColor` | `"#6264A7"` (Teams purple) |
| `copilotAgents.declarativeAgents[0].file` | `"declarative-agent.json"` |
| `permissions` | `["identity", "messageTeamMembers"]` |
| `validDomains` | `[]` |

## Deployment

### Prerequisites

- Microsoft 365 tenant with Copilot license
- Admin access to Teams Admin Center or Developer Portal

### Steps

1. Package the output directory as a `.zip` file:
   ```bash
   zip -j agent-package.zip declarative-agent.json manifest.json color.png outline.png
   ```

2. Upload via Teams Admin Center or Microsoft 365 Developer Portal:
   - Navigate to **Teams apps > Manage apps > Upload new app**
   - Upload the `.zip` package

3. Alternatively, use the Teams Toolkit (VS Code extension) for local sideloading:
   ```bash
   teamsapp package
   teamsapp deploy
   ```

### Icon Requirements

The manifest references `color.png` (192×192 px) and `outline.png` (32×32 px). These are not
generated by AgentShift — provide your own or use Teams Toolkit defaults.

## Known Limitations

| Feature | Status |
|---------|--------|
| Instructions | FULL (8,000 char limit — may truncate) |
| Shell tools | NONE — dropped, unsupported by M365 platform |
| MCP tools | PARTIAL — only Teams/Email/Graph map; others dropped |
| Knowledge (URL) | PARTIAL — URL-based WebSearch supported |
| Knowledge (files) | NONE — local files dropped |
| Cron triggers | NONE — no scheduled triggers in declarative agents |
| Delivery triggers | NONE — no proactive delivery support |
| Constraints | STUB — no direct OS/binary constraint equivalent |
