## Overview

The OpenAI Agents SDK (`pip install openai-agents`) is a lightweight, provider-agnostic Python
framework for building multi-agent AI workflows. Released in early 2025 as the production successor
to the experimental "Swarm" prototype.

Unlike Bedrock or Vertex AI (which use JSON/YAML config files), OpenAI Agents SDK is **code-first**:
agents are defined as Python objects. The emitter generates a ready-to-run Python module, not a
static config file.

GitHub: https://github.com/openai/openai-agents-python
Docs:   https://openai.github.io/openai-agents-python

## Agent Definition (Python)

```python
from agents import Agent, Runner, function_tool

agent = Agent(
    name="My Agent",
    instructions="System prompt text goes here.",
    model="gpt-4o",                # Optional; defaults to gpt-4o
    tools=[my_tool, other_tool],   # @function_tool-decorated callables
    handoffs=[],                   # List of other Agent objects for delegation
    output_type=None,              # Optional Pydantic BaseModel for structured output
)
```

### Key `Agent` Parameters

| Parameter      | Type                          | Required | Notes |
|----------------|-------------------------------|----------|-------|
| `name`         | `str`                         | вњ…       | Human-readable identifier |
| `instructions` | `str` or `Callable`           | вњ…       | System prompt; can be dynamic |
| `model`        | `str`                         | вќЊ       | e.g. `"gpt-4o"`, `"gpt-4o-mini"` |
| `tools`        | `list[FunctionTool]`          | вќЊ       | `@function_tool`-decorated functions |
| `handoffs`     | `list[Agent]`                 | вќЊ       | Agents this agent can delegate to |
| `output_type`  | `type[BaseModel]`             | вќЊ       | Pydantic model for structured output |

## Tool Definition (`@function_tool`)

```python
from agents import function_tool

@function_tool
def get_weather(city: str) -> str:
    """Get current weather for a city."""
    return f"Weather in {city}: sunny"
```

- Decorator reads the Python docstring as the tool description
- Parameter types are auto-converted to JSON Schema via type hints
- Supports `async def` tools natively

## IR в†’ OpenAI Agents SDK Mapping

| IR Field | OpenAI Agents SDK | Notes |
|----------|-------------------|-------|
| `identity.name` | `Agent(name=...)` | Direct |
| `identity.description` | Module docstring | Informational |
| `instructions` | `Agent(instructions=...)` | Direct; no char limit |
| `tools[].name` | `@function_tool` function name | snake_case conversion |
| `tools[].description` | Function docstring | Used by SDK for tool description |
| `tools[].parameters` | Function type hints | Converted from JSON Schema |
| `model` | `Agent(model=...)` | Default: `"gpt-4o"` |
| `persona.sections` | Prepended to instructions | Sections joined with headers |
| `triggers[]` | README note | SDK has no native trigger system |
| `knowledge[]` | README note + comment | No native knowledge store |
| `governance.guardrails` | Inline comment stubs | SDK supports guardrails via Runner |

## Multi-Agent Handoffs (Advanced)

When an IR has multiple agents (e.g. from AGENTS.md or multi-agent registry export), the emitter
can generate a triage pattern:

```python
specialist = Agent(name="Specialist", instructions="...")
triage = Agent(
    name="Triage",
    instructions="Route requests to the right specialist.",
    handoffs=[specialist],
)
```

For single-agent IR (the common case), `handoffs=[]`.

## Platform Constraints

| Constraint | Detail |
|------------|--------|
| No config file format | Agents are code, not config. Emitter generates `.py` files. |
| Model must be OpenAI | SDK defaults to OpenAI API. Other providers need adapters. |
| No native cron/trigger | Must integrate with external scheduler. |
| No native knowledge base | Must implement retrieval in tool functions. |
| Tool params via type hints | Complex nested objects need Pydantic models. |
